思路

无法模拟登陆过程，所以无法本机debug

https://developers.weixin.qq.com/miniprogram/dev/api/signature.html#wxchecksessionobject
但是感觉服务器可伪造

  1. 不同应用程序中是否存在相同的 miniSession 应该是类似于LocalStorage的储存，跨域不可读。是否能存在类似xss，读取高权限用户的miniSession。

  2. 通过伪造code iv等信息发现code的生成是有appid做seed的（猜测）

    测试： 抓取不同小程序的code看看是否能够混用，如果能够混用说明是debug环境有此校验

    check接口的状态码的判别

    1) 向login/login.json请求code、encryptedData、iv、version => 向微信api.weixin.qq.com/sns/jscode2session置换session_key、openid ,通过openid去判断在login上的userid 向client返回userid和sessionid

    2) client存userid和sessionid到localstorage，再次向mini/ request的时候直接用sessionid
    Q1:服务器如何判断sessionid是否有效
    猜测:1. login 和 mini 两个域在同一服务器上 但是个人并不认为能完成session的传递

    2. mini上有传递sessionid的接口。
  1. 全局审计代码，找是否有传递敏感数据的接口
  2. 能否伪造sessionid让mini识别
  3. login server 必然会发起请求，审计spring boot是否存在ssrf
  4. 使用xpost动态debug
  5. bypass domain whitelist
